# Implementation Patterns Plugin

Proven patterns for implementing systems: design patterns, data structures, algorithms, and architectural patterns.

## Skills

1. **design-patterns** - Gang of Four patterns (Creational, Structural, Behavioral)
2. **data-structure-selection** - Choosing optimal data structures for use cases
3. **algorithm-analysis** - Time/space complexity, Big O notation
4. **concurrency-patterns** - Actor model, CSP, lock-free, async/await
5. **functional-patterns** - Higher-order functions, composition, monads
6. **reactive-patterns** - Streams, observables, event-driven reactive systems
7. **state-management** - Finite state machines, event sourcing, CQRS
8. **resource-management** - RAII, cleanup patterns, resource leaks, lifetime management

## Commands

1. **implement-pattern** - Choose and implement appropriate pattern for problem
2. **optimize-code** - Analyze algorithms and select optimal implementation
3. **design-solution** - System design combining multiple patterns

## Standards Reference

- Gang of Four, _Design Patterns: Elements of Reusable Object-Oriented Software_
- Donald Knuth, _The Art of Computer Programming_
- Reactive Manifesto (Ellis, Kuhn, Meyarivan, Vinoski)
