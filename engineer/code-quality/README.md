# Code Quality Plugin

Comprehensive guide to writing clean, maintainable, and correct code through systematic refactoring, SOLID principles, code review, and quality metrics.

## Skills

1. **clean-code-review** - Systematic code review checklist for readability, maintainability, and correctness
2. **refactoring-catalog** - Common refactoring patterns and when to apply them
3. **solid-principles** - Single Responsibility, Open/Closed, Liskov, Interface Segregation, Dependency Inversion
4. **code-smell-detector** - Identifying long methods, duplication, low cohesion, high coupling
5. **naming-conventions** - Choosing meaningful names for functions, variables, classes, and modules
6. **function-design** - Pure functions, single purpose, side effect management
7. **error-handling-patterns** - Exceptions, error types, recovery strategies
8. **code-complexity-analysis** - Cyclomatic complexity, cognitive complexity metrics
9. **dependency-injection** - Loosely coupled, testable code through DI patterns
10. **interface-design** - Minimal, cohesive, role-based interfaces

## Commands

1. **review-code** - Chain clean-code-review + code-smell-detector for comprehensive analysis
2. **refactor** - Chain refactoring-catalog + dependency-injection for safe refactoring
3. **assess-quality** - Chain code-complexity-analysis + naming-conventions + error-handling-patterns
4. **apply-solid** - Chain all SOLID-related skills for principled design

## Standards Reference

- Robert C. Martin's _Clean Code_ and _The Clean Coder_
- Martin Fowler's _Refactoring_ (2nd edition) and code smell catalog
- Michael Feathers' _Working Effectively with Legacy Code_
- Sandi Metz' _Practical Object-Oriented Design_
