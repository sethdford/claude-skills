1. Use the `solid-principles` skill to identify SOLID violations (which principle is violated?)
2. Use the `interface-design` skill to assess and redesign interfaces for cohesion and minimalism
3. Use the `function-design` skill to ensure functions are single-purpose and testable
4. Synthesize refactoring strategy: which principle to address first, which changes are safe?

After completion, suggest: "Would you like me to execute this refactoring plan, or should we review it with the team first?"
