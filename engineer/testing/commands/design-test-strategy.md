Chain these steps:

1. Use the `test-strategy` skill to design pyramid (unit/integration/E2E split)
2. Use the `integration-test-design` skill to identify key component interactions
3. Use the `test-data-management` skill to plan data infrastructure
4. Synthesize into a written test plan with specific tests to write

After completion, suggest: "Next, use the `write-tests` command to implement the plan."
