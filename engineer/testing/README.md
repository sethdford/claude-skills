# Testing Plugin

Comprehensive testing strategy and execution from test planning through mutation testing and test data management.

## Skills

1. **test-strategy** - Overall testing plan: pyramid, coverage goals, layers
2. **tdd-workflow** - Red-Green-Refactor cycle, test-first development
3. **unit-test-design** - Single-unit tests, isolated behavior, mocking strategy
4. **integration-test-design** - Multi-component tests, contract verification
5. **property-based-testing** - QuickCheck, Hypothesis, generative testing
6. **test-doubles-strategy** - Mocks, stubs, fakes, spies; when to use each
7. **mutation-testing** - Measuring test quality via mutation injection
8. **test-data-management** - Fixtures, factories, realistic test data
9. **performance-test-plan** - Load testing, profiling, SLA verification

## Commands

1. **write-tests** - Design and write unit tests using TDD workflow
2. **design-test-strategy** - Create pyramid and coverage strategy for new feature
3. **validate-coverage** - Check coverage and mutate tests to ensure real verification
4. **test-plan** - Full integration test plan for component interaction

## Standards Reference

- Kent Beck, _Test-Driven Development: By Example_
- Michael Feathers, _Working Effectively with Legacy Code_
- Martin Fowler, _Microtest Guide_ and _TestPyramid_
- Roy Osherove, _The Art of Unit Testing_
