# Engineer Toolkit Plugin

Essential cross-cutting skills for effective engineering: documentation, collaboration, processes, and incident management.

## Skills

1. **technical-writing** - Clear technical docs, API docs, architecture decision records
2. **git-workflow** - Feature branches, commit hygiene, rebasing, merging strategies
3. **code-review-checklist** - Systematic code review, feedback, approval criteria
4. **pair-programming-guide** - Effective pairing, driver/navigator roles, tools
5. **on-call-runbook** - Runbook templates, escalation procedures, incident playbooks
6. **incident-response** - SEV classification, incident command system, postmortems
7. **documentation-strategy** - Documentation levels (README, architecture, API, runbooks)

## Commands

1. **write-docs** - Create technical documentation with examples and structure
2. **setup-workflow** - Define team git workflow and process
3. **incident-response** - Start incident response with classification and response

## Standards Reference

- William Strunk Jr., _The Elements of Style_
- Google Technical Writing Course
- Site Reliability Engineering (SRE) book and practices
