# Engineering Excellence Plugin

Establish standards, improve developer experience, and build a culture of continuous improvement.

## Skills (8 skills)

- **coding-standards** — Define and enforce coding standards that improve readability and reduce defects
- **developer-experience-audit** — Audit and improve developer experience (DX) for productivity
- **tooling-strategy** — Strategically select and configure tools that enable velocity
- **documentation-standards** — Create documentation practices that scale with team growth
- **testing-strategy** — Balance unit, integration, and end-to-end testing for confidence and speed
- **monitoring-strategy** — Design observability and monitoring that enables rapid diagnosis
- **security-practices** — Build security into the development process, not as an afterthought
- **accessibility-standards** — Ensure products are accessible to all users

## Commands (3 commands)

- **audit-dx** — Identify developer experience friction points and plan improvements
- **define-standards** — Create coding and engineering standards
- **improve-tooling** — Evaluate and recommend tooling improvements

## Key Principles

1. Standards free up cognitive load: clear rules enable faster decisions
2. Developer experience is a competitive advantage: faster local workflows = faster shipping
3. Testing is insurance: automated tests catch regressions before customers do
4. Observability prevents incidents: good monitoring detects problems early
5. Security and accessibility are non-negotiable: build them in from day one
