# Process Engineering Plugin

Build sustainable development workflows, CI/CD pipelines, release processes, and incident response.

## Skills (8 skills)

- **development-workflow** — Design branching strategies and PR workflows that maximize velocity
- **branching-strategy** — Choose branching models (Git Flow, GitHub Flow, trunk-based) for your team
- **release-management** — Plan releases: versioning, deployment, rollback, communication
- **incident-management-process** — Create blameless incident response processes and postmortems
- **on-call-rotation** — Design sustainable on-call systems that prevent burnout
- **change-management** — Plan and communicate technical changes that affect operations
- **process-improvement** — Systematically identify bottlenecks and iterate on processes
- **definition-of-ready** — Define criteria for work to enter development (prevents rework)

## Commands (3 commands)

- **setup-workflow** — Design and document team development workflow
- **plan-release** — Plan release with versioning, deployment, and rollback strategy
- **improve-process** — Identify bottlenecks and plan process improvements

## Key Principles

1. Workflows should minimize context switching and merge conflicts
2. Automation is cheaper than manual process discipline
3. On-call should be sustainable: pagers shouldn't own your life
4. Incidents are learning opportunities, not blame opportunities
5. Process improvements should be data-driven, not opinion-driven
