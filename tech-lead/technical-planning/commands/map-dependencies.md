---
description: Identify and visualize dependencies across teams and projects to prevent blocked work.
argument-hint: [list of initiatives or epics]
---

Chain these steps:

1. Use the `work-breakdown-structure` skill to decompose initiatives and identify work streams and owners
2. Use the `dependency-mapping` skill to build explicit dependency graph (blocking, resource, informational)
3. Use the `risk-register` skill to identify risks associated with critical path items
4. Synthesize findings into a dependency diagram with critical path highlighted and mitigation strategies for top risks

After completion, suggest:

- How to communicate critical path to teams and set expectations
- What handoff points need scheduling and acceptance criteria definition
- How to run contingency planning for high-risk critical path items
