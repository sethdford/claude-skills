# Technical Decision-Making Plugin

Master RFCs, technology evaluation, and architecture decisions at scale.

## Skills (8 skills)

- **rfc-process** — Structure RFCs that drive collaborative technical decisions
- **technology-evaluation** — Systematically evaluate tech choices using frameworks and trade-off analysis
- **build-vs-buy-analysis** — Decide when to build solutions internally vs. buy/integrate
- **spike-plan** — Design investigation tasks that reduce architectural uncertainty
- **proof-of-concept-criteria** — Define what makes a POC successful before building
- **decision-matrix** — Build frameworks to compare options quantitatively
- **risk-mitigation-plan** — Identify and mitigate risks in major technology decisions
- **post-decision-review** — Learn from decisions by measuring outcomes vs. expectations

## Commands (3 commands)

- **write-rfc** — Create comprehensive RFC with alternatives and tradeoffs
- **evaluate-technology** — Systematically evaluate and recommend technology choices
- **plan-spike** — Design investigation task to de-risk technical uncertainty

## Key Principles

1. Document decisions: written RFCs enable async review and future reference
2. Show your work: comparison matrices and trade-off analysis build confidence
3. Explore alternatives: the best decision includes explaining why you rejected other options
4. De-risk through spikes: proof-of-concept before committing to major changes
5. Learn from decisions: measure outcomes against predictions
