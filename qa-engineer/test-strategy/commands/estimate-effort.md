# estimate-effort

Estimate testing effort and resource requirements.

## Chains

1. **test-estimation** — Estimate effort for testing activities
2. **test-strategy-design** — Establish test approach and scope
3. **risk-based-test-plan** — Prioritize testing to align with risk

## Usage

```
/estimate-effort "Describe the feature scope, complexity, and timeline"
```

## Expected Output

- Effort estimates by test activity (design, execution, automation)
- Resource requirements (QA engineers, tools, environments)
- Timeline and milestones
- Confidence level and assumptions
- Recommendations for effort optimization

## Example

```
/estimate-effort "New payment module with 8 new endpoints, complex state machine.
2-week development timeline. Team: 2 junior, 1 senior QA. Need automated + manual testing."
```
