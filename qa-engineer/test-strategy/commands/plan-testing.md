# plan-testing

Design a comprehensive test strategy and plan for a project.

## Chains

1. **test-strategy-design** — Establish overall testing approach and scope
2. **risk-based-test-plan** — Prioritize test areas by risk
3. **test-estimation** — Estimate testing effort and timeline

## Usage

```
/plan-testing "Describe the project scope, timeline, and quality goals"
```

## Expected Output

- Comprehensive test strategy document
- Risk prioritization matrix
- Testing effort estimate
- Test timeline and milestones
- Resource requirements
- Success criteria

## Example

```
/plan-testing "Building a payment processing system with 3-month timeline.
Critical paths: transaction processing, security, compliance.
Budget: 4 QA engineers. Must support GDPR and PCI compliance testing."
```
