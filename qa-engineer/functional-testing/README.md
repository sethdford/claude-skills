# Functional Testing Plugin

Manual testing, exploratory testing, and systematic test case design.

## Overview

This plugin provides guidance for:
- Systematic test case design using multiple techniques
- Exploratory testing using charters and heuristics
- Boundary value analysis for edge case testing
- Equivalence partitioning to reduce test count
- Decision table testing for complex logic
- State transition testing for state machines
- Use case testing for user workflows
- Error guessing for discovery testing

## Skills (8)

| Skill | Purpose |
|-------|---------|
| `test-case-design` | Systematic test case creation |
| `exploratory-testing-charter` | Charter-based exploration |
| `boundary-value-analysis` | BVA and edge case testing |
| `equivalence-partitioning` | Partition test domains |
| `decision-table-testing` | Complex logic validation |
| `state-transition-testing` | State machine testing |
| `use-case-testing` | Scenario-based testing |
| `error-guessing` | Heuristic-based error discovery |

## Commands (3)

| Command | Purpose |
|---------|---------|
| `design-test-cases` | Create comprehensive test cases |
| `explore-feature` | Plan exploratory testing session |
| `analyze-boundaries` | Identify boundary test cases |

## When to Use

- Designing manual test cases for new features
- Exploring features to discover unplanned issues
- Testing edge cases and boundary conditions
- Testing complex business logic and workflows
- Designing efficient test cases with minimum redundancy
