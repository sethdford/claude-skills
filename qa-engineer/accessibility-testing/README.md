# Accessibility Testing Plugin

WCAG compliance, assistive technology testing, and accessibility auditing.

## Overview

This plugin provides guidance for:
- WCAG 2.2 compliance auditing
- Screen reader and assistive technology testing
- Keyboard navigation validation
- Color contrast analysis
- ARIA attribute testing and validation
- Accessibility testing strategy
- Accessibility regression testing

## Skills (7)

| Skill | Purpose |
|-------|---------|
| `wcag-audit` | WCAG 2.2 compliance auditing |
| `screen-reader-testing` | Screen reader testing |
| `keyboard-navigation-testing` | Keyboard accessibility |
| `color-contrast-analysis` | Color contrast validation |
| `aria-validation` | ARIA attribute testing |
| `accessibility-test-plan` | Accessibility testing strategy |
| `accessibility-regression` | Regression testing for accessibility |

## Commands (3)

| Command | Purpose |
|---------|---------|
| `audit-accessibility` | Perform WCAG audit |
| `test-screen-reader` | Plan screen reader testing |
| `validate-wcag` | Validate WCAG compliance |

## When to Use

- Auditing web applications for WCAG compliance
- Testing with screen readers
- Validating keyboard accessibility
- Ensuring accessible colors and contrast
- Testing ARIA implementations
- Preventing accessibility regressions

## Standards Grounding

- WCAG 2.2 Web Content Accessibility Guidelines
- WCAG 2.1 Implementation Guide
- Accessibility best practices
