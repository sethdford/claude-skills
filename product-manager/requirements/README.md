# Requirements Plugin

PRDs, user stories, acceptance criteria, and technical specifications.

## Skills (9)

1. **prd-template** — Structure comprehensive PRD
2. **user-story-mapping** — Map stories across workflows
3. **acceptance-criteria** — Define testable success
4. **jobs-to-be-done-pm** — Frame features around customer jobs
5. **feature-specification** — Write unambiguous feature specs
6. **epic-breakdown** — Decompose into shippable stories
7. **definition-of-done** — Define completion standards
8. **edge-case-analysis** — Systematically identify edge cases
9. **requirements-traceability** — Link requirements through execution

## Commands (4)

1. **write-prd** — Create comprehensive PRD
2. **map-stories** — Map stories across workflows
3. **spec-feature** — Write detailed feature specification
4. **define-epic** — Decompose epic into stories

## Key Concepts

- **Clear requirements prevent rework**: Ambiguity costs engineering time
- **User-centered**: Frame in customer context
- **Testable criteria**: Definition of done is measurable
- **Edge case documentation**: Happy path isn't enough
