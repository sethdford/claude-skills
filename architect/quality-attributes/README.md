# Quality Attributes Plugin

Master non-functional requirements, trade-off analysis, and architectural fitness functions. Design for scalability, reliability, performance, security, and maintainability.

## Skills

- **scalability-analysis** — Model growth, identify bottlenecks, project infrastructure costs
- **reliability-design** — Design fault-tolerant systems with graceful degradation and automatic recovery
- **performance-modeling** — Build performance models to predict latency and throughput
- **security-architecture** — Design layered security with authentication, authorization, encryption
- **maintainability-assessment** — Evaluate code quality, coupling, and testability
- **observability-design** — Design metrics, logs, traces for understanding production systems
- **cost-optimization** — Reduce infrastructure costs while maintaining SLAs
- **trade-off-analysis** — Systematically compare design options and justify choices

## Commands

### assess-quality

Evaluate system across all quality attributes. Uses scalability-analysis, reliability-design, performance-modeling, security-architecture to produce audit.

**Usage**: `assess-quality [system description]`

### design-for-scale

Plan for growth and capacity. Applies scalability-analysis, performance-modeling, cost-optimization to model scaling scenarios.

**Usage**: `design-for-scale [current system and growth projections]`

### trade-off-matrix

Compare design options systematically. Uses trade-off-analysis to create weighted decision matrix and recommendation.

**Usage**: `trade-off-matrix [option descriptions and criteria]`

## When to Use This Plugin

- Evaluating non-functional requirements
- Capacity planning and growth modeling
- Security and reliability audits
- Performance optimization and profiling
- Cost reduction initiatives
- Comparing architectural options
- Building observability and monitoring strategies
