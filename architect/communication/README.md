# Communication Plugin

Master architecture communication through C4 diagrams, RFCs, presentations, and documentation. Align stakeholders and explain decisions.

## Skills

- **c4-model-diagram** — Create C4 context, container, component, and code diagrams
- **architecture-brief** — Write executive-friendly architecture summaries
- **rfc-template** — Propose architectural changes with comprehensive RFCs
- **stakeholder-presentation** — Present architecture to diverse audiences
- **architecture-roadmap** — Create multi-quarter architecture vision and roadmap
- **context-diagram** — Design high-level system boundary diagrams
- **sequence-diagram** — Show interactions and flows over time

## Commands

### document-architecture

Create comprehensive architecture documentation with C4 diagrams and briefs.

### write-rfc

Propose architectural change with detailed RFC and impact analysis.

### present-architecture

Prepare architecture presentation tailored to audience.

## When to Use

- Documenting architecture decisions
- Communicating with non-technical stakeholders
- Proposing major architectural changes (RFCs)
- Creating architecture diagrams
- Presenting at reviews or conferences
- Building shared understanding across teams
