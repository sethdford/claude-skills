# Data Architecture Plugin

Master data modeling, storage selection, data flow design, and data governance. Build scalable, maintainable data architectures.

## Skills

- **data-model-design** — Design data models using normalization and domain-driven principles
- **storage-selection** — Choose databases: relational, NoSQL, data warehouse, search engines
- **data-flow-diagram** — Design data movement and transformation pipelines
- **integration-patterns** — Integrate data across systems via ETL, CDC, event streaming
- **event-sourcing** — Store state as immutable event log for auditability
- **data-governance** — Define data ownership, quality, and compliance policies
- **schema-evolution** — Manage schema changes while maintaining backwards compatibility
- **data-pipeline-design** — Design batch and streaming data pipelines

## Commands

### design-data-layer

Design data architecture for new system. Uses data-model-design, storage-selection, data-flow-diagram.

### model-domain

Create domain data model using DDD and normalization principles.

### plan-integration

Design data integration between systems using integration-patterns and event-sourcing.

## When to Use

- Designing new data architectures
- Selecting databases and storage systems
- Building data pipelines and ETL
- Implementing data governance and compliance
- Managing schema evolution
- Auditing and optimizing data flows
